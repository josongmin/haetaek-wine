# 의존성 설치 가이드

## 프로젝트 구조
```
uxight-admin/
├── apps/
│   ├── client/        # 프론트엔드
│   └── server/        # 백엔드
├── packages/
│   └── shared/        # 공통 패키지
└── package.json       # 루트 workspace
```

## 전체 의존성 설치 (추천)

```bash
npm install
```
루트에서 실행하면 모든 워크스페이스의 의존성을 한번에 설치합니다.

## 특정 워크스페이스에 패키지 추가

### 서버(백엔드)에 추가
```bash
npm install [패키지명] -w apps/server
```

예시:
```bash
npm install express -w apps/server
npm install lodash -w apps/server
```

### 클라이언트(프론트엔드)에 추가
```bash
npm install [패키지명] -w apps/client
```

예시:
```bash
npm install axios -w apps/client
npm install react-router-dom -w apps/client
```

### 공통 패키지에 추가
```bash
npm install [패키지명] -w packages/shared
```

## 개발 의존성 추가

### 루트에 추가
```bash
npm install [패키지명] -D
```

예시:
```bash
npm install concurrently -D
```

### 특정 워크스페이스에 개발 의존성 추가
```bash
npm install [패키지명] -D -w apps/server
```

## 직접 워크스페이스에서 설치

### 서버
```bash
cd apps/server
npm install
```

### 클라이언트
```bash
cd apps/client
npm install
```

## 의존성 정리 및 재설치

### node_modules 전체 삭제 후 재설치
```bash
# node_modules 삭제
rm -rf node_modules apps/*/node_modules packages/*/node_modules

# 재설치
npm install
```

### package-lock.json 초기화
```bash
rm -f package-lock.json
npm install
```

## 자주 사용하는 명령어

### 의존성 버전 확인
```bash
npm list [패키지명]
```

### 특정 워크스페이스 의존성 확인
```bash
npm list -w apps/server
```

### 업데이트 가능한 패키지 확인
```bash
npm outdated
```

### 보안 취약점 확인
```bash
npm audit
```

### 보안 취약점 자동 수정
```bash
npm audit fix
```

## 주의사항

1. 워크스페이스 구조에서는 가급적 루트에서 `-w` 옵션을 사용
2. 직접 워크스페이스로 이동해서 설치하면 루트 package-lock.json과 충돌 가능
3. node 버전이 맞지 않으면 경고가 나올 수 있음 (현재 요구: 20.x, 실제: 22.x)
